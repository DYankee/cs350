18.19

A PARTS file with Part# as the key field includes records with the following Part# values:
23, 65, 37, 60, 46, 92, 48, 71, 56, 59, 18, 21, 10, 74, 78, 15, 16, 20, 24, 28, 39, 43, 47, 50, 69, 75, 8, 49, 33, 38.
Suppose that the search field values are inserted in the given order in a B+-tree of order p = 4 and pleaf = 3; show how the tree will expand and what the final tree will look like.

Because its hard to illustrate the links between nodes assume the following pointers exist in each node:

  • All leaf nodes have at most 3 page numbers and a pointer to the node to their right (node with next highest page number)
  • All internal nodes have at least 2 pointers and at most 4
  • Internal node pointers behave in the following ways:
    • Left pointer: all page numbers < the first value in the node
    • Middle or middle Left pointer: all page numbers >= the first value and less then the second value in the node
    • Middle right pointer: all page numbers >= the second value and less then the third value in the node
    • right pointer: all page numbers >= the third value in the node
